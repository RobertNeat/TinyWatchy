# Code structure

Application sources live in `src/`, source assets in `resources/`, PlatformIO configuration in `platformio.ini`, and build-time compatibility scripts in `scripts/`.

## Core components

- `main.cpp`: Arduino entry point that starts `TinyWatchy`.
- `TinyWatchy.cpp` / `TinyWatchy.h`: hardware initialization, wake handling, data updates, NTP scheduling, and deep sleep.
- `AlarmHandler.cpp` / `AlarmHandler.h`: system and user alarms, RTC wake scheduling, and active/sleep accelerometer state.
- `NTP.cpp` / `NTP.h`: NTP synchronization and RTC updates.
- `WiFiHelper.h`: short-lived Wi-Fi connections used by NTP.
- `Menu.cpp` / `Menu.h`: button handling, menu navigation, and option ownership.
- `Screen.cpp` / `Screen.h`: watch-face registration, selection, and rendering.
- `ScreenInfo.h`: data passed to every watch face.
- `DateTime.h`: compact time structure compatible with the RTC conversion code.
- `defines.h`: hardware constants, private-configuration propagation, and compile-time validation.
- `defines_private.h.template`: tracked template for per-device configuration.
- `defines_private.h`: local configuration; generated by the user and ignored by Git.
- `resources.h`: generated font and bitmap data; ignored by Git.
- `Watchy/`: display code, icons, and the BMA423 driver derived from the original Watchy firmware.

## Extension points

- `Faces/`: implementations of `AbstractFace`. See [Adding a watch face](Adding_watchface.md).
- `MenuOptions/`: implementations of `AbstractOption`. See [Adding a menu option](Adding_option.md).
- `resources/fonts/source-fonts/`: source font files and their licenses.
- `resources/images/`: source images for generated bitmaps.

## Build configuration

`platformio.ini` pins the platform and library versions for reproducible builds. `scripts/patch_dependencies.py` applies a version-checked const-correctness fix to the pinned `Rtc_Pcf8563` 1.0.3 source before compilation. The script fails instead of silently applying an unknown patch if the dependency no longer matches the expected version.

PlatformIO generates `.pio/`, `.vscode/c_cpp_properties.json`, and `.vscode/launch.json` for the current machine. These files must not be copied between systems or committed.
